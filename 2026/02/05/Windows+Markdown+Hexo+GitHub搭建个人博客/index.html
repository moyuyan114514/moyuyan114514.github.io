了解 Hexo：你的数字图书馆管理员

先贴上 Hexo 的官方链接：Hexo 中文文档。

你可以把 Hexo 理解为一个极其勤快的图书管理员：你只需要负责用 Markdown 格式写下你的思想，它负责按照你选好的样式进行排版，最后印刷成网页。

为什么选择 Hexo？

  • 内容主权：脱离社交网站，内容安全掌握在自己手里。
  • 主题丰富：拥有大量开源主题，审美随心换。
  • 静态安全：纯网页博客，服务器负载低，且能白嫖 GitHub Pages 实现免费访问。
  • 数据自由：从配置到内容都在 Markdown 文件中，随时可以打包带走。

必经的挑战

  • 环境配置：需要在电脑上安装 Node.js 和 Git。
  • 技能学习：需要掌握基础的 Markdown 语法和简单的 Git 指令。

第一阶段：环境搭建

1. GitHub 账户与仓库

  1. 注册：访问 GitHub 官网 注册账号。

  2. 创建仓库：仓库名必须为 你的用户名.github.io。

2. 必备软件安装

  • Node.js：Node.js — 下载选择 Windows 下载并一路 Next 安装。

  • Git：下载并安装 Git 官网 程序，保持默认设置。

3. 配置 SSH 公钥

为了避免每次上传博客都要输入 GitHub 账号密码，必须配置 SSH：

  1. 在桌面右键选择 Git Bash Here。

  2. 生成密钥：输入 ssh-keygen -t rsa -C "你的邮箱"，连续按三下回车。

  3. 获取密钥：输入 cat ~/.ssh/id_rsa.pub，复制显示的以 ssh-rsa 开头的所有代码。

  4. 绑定 GitHub：进入 GitHub Settings -> SSH and GPG keys -> New SSH key，将代码粘贴进去并保存。

  5. 验证密钥：在 Git Bash 输入：ssh -T git@github.com。
    如果看到 Hi [你的用户名]! You've successfully authenticated... 说明配置无误。

3. 常用git指令

在博客文件夹下运行：

  1. hexo n "我的第一篇文章"：创建新文章。

  2. hexo s：本地预览（在浏览器输入 localhost:4000 提前查看）。

  3. hexo clean：清除缓存。

  4. hexo g：生成静态网页。

  5. hexo d：一键部署到 GitHub（正式发布）。
